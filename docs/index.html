readable-or-else

A readability CI gate — or else, a rewrite

readable-or-else is a readability CI gate — a Flesch-Kincaid grade level check in CI that fails a pull request when a page misses its target reading level — plus an LLM-backed mode that can rewrite text to a 7th grade reading level with an LLM (or any grade you choose) so a failing build doesn't stall on “now what.” It's built for plain language and accessibility work, including a ready-made WCAG 3.1.5 Reading Level preset, and ships general-purpose: point it at any HTML or text content, not just civic sites.

Measurement is a thin, honest wrapper over textstat's Flesch-Kincaid formula — readable-or-else doesn't reimplement the math, only the policy around it (gate/warn/ratchet, presets, the rewrite loop). Every rewrite below is a real call to an LLM, re-measured with that same formula, and checked against every preset the tool ships. Source on GitHub →

Three named presets ship with readable-or-else, each carrying its citation as data, not a hardcoded assumption (see readable_or_else/presets.py):
nycsg7
target grade ≤ 7
NYC Web Content Style Guide, "Reading level": designsystem.nyc.gov/standards/nyc-web-content-style-guide.html — “Try to simplify your content to a seventh grade reading level or lower. We use the Flesch–Kincaid grade level formula.”
govuk9
target grade ≤ 4
GOV.UK content design, “Writing for GOV.UK” — targets a reading age of 9. textstat has no reading-age formula, so this preset approximates reading age 9 as Flesch-Kincaid US grade ≤4 (the common age-minus-5 heuristic).
wcag-aaa
target grade ≤ 9 (flag-only)
WCAG 3.1.5 Reading Level (AAA): w3.org/WAI/WCAG22/Understanding/reading-level.html — requires a simplified supplement above lower-secondary reading level, not a hard numeric gate. readable-or-else models this as flag-only: it never fails a build on this preset.

26 U.S.C. § 61(a) (U.S. Internal Revenue Code — “Gross income defined”)

The flagship example of dense American legalese — a single sentence with a fourteen-item enumerated list, and no rewrite below landed at any preset’s target. Simplifying a definitional list like this one, without dropping a category the statute actually defines, may not be possible at these lengths — a real, reportable limit of the rewrite mode, not a bug.

readable-or-else was scoped against crol-list's own pages as its first real corpus. See the ten genuinely-hardest passages from its about.html and data.html, measured and rewritten live against the nycsg7 preset — the full before/after showcase →

Rewrites shown on this page are LLM suggestions, not accepted edits. Each is re-measured with the same Flesch-Kincaid formula used for gating, and checked for meaning preservation (numbers, URLs, and proper nouns in the original must also appear in the candidate) — a heuristic, not a semantic-equivalence proof. A human should read every rewrite before it ships. --suggest never auto-applies a rewrite to a file. See readable-or-else on GitHub for the full tool, its limits, and its test suite.
